切换目录
  • cd /home
    root用户的家目录就是/root，当我们创建了其它用户，就是/home/用户这个目录。

回答家目录的三种方式：
1、cd 然后回车
2、cd ~
3、cd /root或者/home/XXX

cd - ： 表示回到上一次操作的目录。
cd …/ 回退到上一层目录，
cd …/…/ 回退到上两层目录。

对于目录的理解：
相当于是windows上文件夹的意思。

绝对路径和相对路径：

绝对路径：代表的是从/根目录开始；以根目录开始：/home/hadoop/data/ruozeinput.txt
相对路径：不从根目录为开始，以当前光标所在的目录开始，eg：我们进入到/home/hadoop目录下

相对路径的表达：
只要不是以根目录开始就是相对路径的表达：

清屏
  • clear
ls查看当前光标所在的目录文件有哪些

ls：查看当前目录下的文件
ls -l：查看所有文件（不包括隐藏文件）
ls -l -a：查看所有文件（包括隐藏文件）

注意：

  • 在我们的生产上，也可以将一些配置文件放在隐藏文件下。
查看文件的大小

1、ll -h -h, --human-readable with -l, print sizes in human readable format
(e.g., 1K 234M 2G)

根据时间修改先后顺序来对文件排序：

  • ll -rt
    -r reverse(反向) order while sorting
    -t sort by modification time

应用场景：

  • 在生产配置中，不知道哪个文件最新被使用的

mv和cp

mv移动ruoze.log文件到/home/hadoop/a文件夹下
[hadoop@hadoop001 ~]$ mv ruoze.log /home/hadoop/a

mv在移动文件的时候重命名？

  • [hadoop@hadoop001 a]$ mv ruoze.log /home/hadoop/emp.txt

仅在当前目录修改名称？

  • mv emp.txt test.txt

拷贝文件：cp
问题：mv快还是cp块？

  • mv始终只有一份，相当于只是修改了链接指向。
  • 而cp比较慢

拷贝文件

在当前路径下拷贝文件：
[hadoop@hadoop001 ~]$ cp test.txt test1.txt

当前路径下拷贝文件到其它目录顺便改名字：
[hadoop@hadoop001 ~]$ cp test.txt /home/hadoop/a/ruoze.log

如何创建一个空文件？

touch rz.log
echo “” >> rz1.log
在生产上创建空文件或者把文件内容置为空，echo慎用；因为这个echo还是占一个字节的。cat去查看内容其实都是没有东西，其实占着一个空位符。

生产场景：shell脚本定时调度去清空，判断内容是不是0字节，其实echo是有1个字节，那这样就是有问题的。

比如一个文件中是有内容的，如何去进行置空？
cat /dev/null > rz1.log，此时置空也是0字节。

是覆盖，>>是追加。

查看文件内容？

cat 文件内容一下子全部显示在控制台
more 文件内容一页一页的往下翻，按空格键往下翻，无法回退，按q键退出。
less 文件内容往上 往下都能查看，按上下箭头的按键，按q键退出。

比如一个文件日志100M的日志内容，如何去查看？
配置文件：内容较少，cat more less都可以使用；

log日志，当内容较多的时候，进行实时查看：tail -f xxx.log

tail -f和tail -F的区别？

生产举例：
XXX.log是log4j代码项目配置输出日志，正常规则：输出XXX.log，对这个XXX.log保留10份，比如aaa服务写每份日志每份规格是100M。

aaa --> XXX.log 一下子写到99M
XXX.log 当他写到100M的时候，把XXX.log使用mv重命名为XXX.log1，然后touch了一个空的文件还是叫XXX.log。

我们进行的操作：

  • mv ruoze.log ruoze.log1
  • touch ruoze.log
  • echo “ruozedata” >> ruoze.log

进行如上操作后，tail -f已经失去了追踪文件的能力，tail -F还有一个retry的功能，当我们删除又重新创建相同文件的时候，F能嗅探到。

大数据环境中的很多组件定位日志，比如部署CDH，服务没启动，需要自己去进行排查。

当我们出现error的时候需要自己去定位，error文件很小的时候，把日志上传到windows，用editplus进行打开编辑。

cd /var/log，正常上传下载需要安装命令：yum install lrzsz
使用sz boot.log下载到本地，具体下载到本地的那个位置看我们的配置：

  • 如上只适用于文件比较小的，几十兆。

但是对于文件太大的，学会一个命令：

  • cat boot.log | grep ERROR 只打印当前error行数

  • 管道符：前面的输出结果作为后面的输入结果。

  • cat boot.log | grep -A 10 ERROR error的后十行

  • cat boot.log | grep -B 10 ERROR error的前十行

  • cat boot.log | grep -C 10 ERROR error的前后十行，经常用的命令，来定位ERROR的上下文。

windows上文件上传到linux上，使用命令rz；
linux上文件下载到windows上，使用命令sz.

flume针对一般日志采集的方式在生产上一般使用Exec source，使用tail -F；否则mv移动之后使用touch新建的文件是不能显示的。

J总的习惯：
tail -200f XXX.log，实时查看倒数300行文件

tail -30F XXX.log提示语法错误
