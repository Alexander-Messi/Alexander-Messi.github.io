char的话即使只输入一个a，但是它也会自动补全为255个字节；会造成资源浪费。
varchar的话输入一个a，实际一个字节存储也是占据一个字节，不会造成资源浪费。

  • 日期类型

timestamp 年月日时分秒 YYYY-MM-DD HH::MM:SS

面试题：datetime和timestamp的区别？
https://www.cnblogs.com/mxwz/p/7520309.html
区别体现在：
1、存储的方式不同：对于timestamp，它会把插入的时间从当前时区转化为UTC（世界标准时间）进行存储。查询时，将其又转化为客户端当前时区进行返回。
对于DATETIME，不做任何改变，就是原样输入原样输出。

两者所能存储的时间范围不一样：
timestamp所能存储的时间范围是：‘1970-01-01 00:00:01.000000’ 到 ‘2038-01-19 03:14:07.999999’；
而datetime所能存储的时间范围为：‘1000-01-01 00:00:00.000000’ 到 ‘9999-12-31 23:59:59.999999’。

除了存储方式和存储范围不一样之外，其余没有太大区别，对于跨时区的业务，TIMESTAMP更为合适。

第三章：SQL建表规范&增删改查

5、我们已经创建了ruozedata这个数据库：

建表语句的规范：
a.表名称、字段名称不能是中文、汉语拼音；
b.统一风格：已经存在的风格是什么，比如create_user/createuser/Create_user，尽量去跟已经存在的表去统一；经验风格 --> lead check
c.风格不统一的问题：ctime/createtime/cre_time/cre_time

MySQL使用自增长主键有什么优势？

  • 第一个字段必然是id自增长，设置为主键，这个id是没有意义的，只是为了能够高性能的查询，懂不懂MySQL的点就看这个。

一张表只能是有一个主键id，自增长主键 ==>unique+not null，在业务字段中需要唯一值该怎么设置，需要我们设置唯一约束，

见下图：在Dbeaver上的操作：
1、点击ruozedata数据库下的表，双击约束，新建约束，设置name为unique_key，点击确定，会跳出一段SQL，我们把它复制下来，执行；注意：直接点击执行会报错。

SQL语句如下：

  • #给ruozedata数据库下的emp表增加一个唯一约束，name:
    ALTER TABLE ruozedata.emp
    ADD CONSTRAINT emp_un UNIQUE KEY (name);

  • 此时再添加重复用户就会有提醒：
    #给ruozedata数据库下的emp表增加一个唯一约束，name:
    ALTER TABLE ruozedata.emp
    ADD CONSTRAINT emp_un UNIQUE KEY (name);

规范：建表的时候添加注释：
步骤：双击需要选中的表，点击属性，点击列，添加业务字段的相关注释，切记保存。

如何在建表的时候加上注释，COMMENT ‘用户名称’

建表小结：

  • 我们的一张表需要养成规范，头部自增长ID（高性能的查询），中间根据业务需求是否设置unique_key，尾部createuser/create_time和updateuser/update_time建议要加上，表明这条记录在什么时候被创建的，什么时候被修改的，便于修改和追踪，别以后要查询，无从查起。

第六点：增删改查：
#查询
select * from emp;

#插入数据
insert into emp(name,age,createuser,updateuser)
values(‘john’,25,‘root’,‘root’);

#删除数据
delete from emp where name=‘john’ and age=25;

#修改数据
update emp set age=23 where name=‘john’;

连续修改两个字段：只要按逗号拼接就行：
update emp set age=27,name=‘Tom’ where id=1;

注意点：
在生产上写SQL运行的时候，切记不要使用*，因为你不知道这个表的数据量有多大，不知道你的这个操作是否会对别人正在运行的SQL有改变。

背景：财务系统，SQL写删除命令，没有写where条件，如何解决，通过MySQL中的binlog文件进行恢复，arch/mysql-bin.00001

未来在工作当中无论是update还是delete，切记where需要加上条件，加上时间范围查询，createtime > ‘2019-01-01 00:00:00’

举例：表数据一亿条数据，select * ，数据库又是生产库，完蛋，服务就夯住了；那如何查看样本数呢，select name,age from emp linit 3; 看前三条样本数据。全量查询是很慢的。

引出概念：MySQL主从复制：
在生产上，主库用来写，从库用来读。
