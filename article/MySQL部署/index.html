如何重新部署MySQL？

1、rm -rf arch/* binlog文件（存放的是二进制数据）
rm -rf data/* 存放数据文件

2、继续在mysqladmin用户下：
scripts/mysql_install_db
–user=mysqladmin
–basedir=/usr/local/mysql
–datadir=/usr/local/mysql/data

3、重新部署完后还是修改用户密码：
重新启动mysql服务报错：
iZuf6idov4pacdnaao7dsfZ:mysqladmin:/usr/local/mysql:>service mysql start
Starting MySQL…The server quit without updating PID file (/usr/local/mysql/data/hostname.pid).[FAILED]
排查原因：

  • 因为执行1 2两步操作时就有mysql的进程在运行，没有停下来导致重新启动冲突了：
    验证这一问题：
    iZuf6idov4pacdnaao7dsfZ:mysqladmin:/usr/local/mysql:>ps -ef|grep mysqld
    mysqlad+ 14437 1 0 Dec05 ? 00:00:00 /bin/sh /usr/local/mysql/bin/mysqld_safe --datadir=/usr/local/mysql/data --pid-file=/usr/local/mysql/data/hostname.pid
    mysqlad+ 15115 14437 0 Dec05 ? 00:00:22 /usr/local/mysql/bin/mysqld --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data --plugin-dir=/usr/local/mysql/lib/plugin --log-error=/usr/local/mysql/data/hostname.err --pid-file=/usr/local/mysql/data/hostname.pid --socket=/usr/local/mysql/data/mysql.sock --port=3306
    mysqlad+ 28713 25833 0 13:32 pts/0 00:00:00 grep --color=auto mysqld

  • 如上操作证实了这一问题，只要把之前mysql服务的子进程杀了就行，kill -9 15115，然后重新启动mysql服务即可。
    iZuf6idov4pacdnaao7dsfZ:mysqladmin:/usr/local/mysql:>kill -9 15115
    iZuf6idov4pacdnaao7dsfZ:mysqladmin:/usr/local/mysql:>ps -ef|grep 15115
    mysqlad+ 28793 25833 0 13:33 pts/0 00:00:00 grep --color=auto 15115
    iZuf6idov4pacdnaao7dsfZ:mysqladmin:/usr/local/mysql:>service mysql start
    Starting MySQL.[ OK ]

binlog文件在/etc/my.cnf中的位置：
log-bin=/usr/local/mysql/arch/mysql-bin
面试题：假如binlog文件满了怎么办？

  • mysql有个机制，默认保留X天，过了之后会自动删除。

innodb_log_file_size = 200M

  • 这个保证了ib_logfile0文件满200M以后会写另一份。

MySQL中常见的ERROR?

  • Access denied for user ‘root’@'localhost ’ using password: NO
  • 密码不对；
